User Guide — How to Use This System

Overview

This panel manages the video production pipeline: FLUX image generation, image selection, and two alternative video generation lanes — Wan2.2 via ComfyUI on PRESTO, or manual Kling generation. Each package card below shows one episode's production state.

Production Workflow

  1. Generate FLUX images — In each package card, the "FLUX local" section generates photorealistic source images from your image prompts. Use "Dry run only" to test without consuming GPU. Click "Generate N FLUX images" to start.
  2. Select images — Click "Select Images" to open the image selector. Choose which FLUX-generated images will be used for video generation. The selection becomes the input set for video generation.
  3. Choose a video generation lane — After selecting images, pick one of two alternative paths:
    • Lane A: PRESTO (ComfyUI / Wan2.2) — The "PRESTO Wan2.2" section submits all pending selected images to ComfyUI on PRESTO. Click "Submit N to PRESTO". The system processes each image one at a time using the 81-frame Wan2.2 workflow (~2.7 second videos). Progress and stdout appear in real-time. You can cancel mid-run.
    • Lane B: Manual Kling generation — Click "Show Video Prompts" to reveal each selected image's image-to-video prompt text with copy buttons. Copy a prompt and paste it into Kling along with the corresponding image for manual video generation.
    Choose one lane per image, or mix lanes across images — but never submit the same image to both lanes simultaneously.
  4. Review generated videos — Collect the output videos from whichever lane(s) you used. PRESTO results appear in the package's presto-results folder; Kling outputs are saved manually to VIDNAS.
  5. Create Resolve assembly — Once videos are generated, click "Create Resolve Assembly" to generate the handoff files for DaVinci Resolve editing.

Button Reference

Generate N FLUX images — Runs FLUX.1 Dev locally (vidnux RTX 5070 Ti) to produce source images from prompts. ~48s per 1080x1920 image.

Select Images — Opens the image selector page to choose which images go to video generation.

Submit N to PRESTO — Sends all pending selected images to ComfyUI on PRESTO (RTX 4090). Processes one at a time via 81-frame Wan2.2 i2v workflow. Observed estimate: ~12 min per video; actual time varies with model load and system state.

Show/Hide Video Prompts — Toggles a panel showing each selected image's image-to-video prompt text with copy-to-clipboard buttons for manual Kling generation. If video-prompts.json is not present, the panel shows a "not prepared" message.

Create Resolve Assembly — Generates handoff files for DaVinci Resolve (resolve-handoff folder).

PRESTO Requirements

ComfyUI must be running on PRESTO at http://192.168.50.187:8188 with the queue empty. The status badges at the top show PRESTO reachability and queue state. One job runs at a time — wait for completion before submitting another package.

Troubleshooting

"PRESTO unreachable" — Check that ComfyUI is running on PRESTO and the network is connected.

"Queue busy" — Another ComfyUI job is running. Wait for it to finish or cancel it from the ComfyUI web UI.

"Job already active" — Only one PRESTO job can run at a time. Wait for the current job to complete or cancel it.

"Not prepared" in Video Prompts — The package has no video-prompts.json file. This file contains image-to-video prompts written for Kling. It must be created before the Video Prompts panel can display anything. The system does not fall back to image-prompts.json (those are text-to-image prompts, not video prompts).
