# Cockpit `/aigen-assets/` Root Fix (2026-07-03)

Follow-up to `reports/cockpit-visual-verify.md` (verdict PARTIAL). Fixes the one
real rendering defect that verification found. Base commit `74ca364` (main),
machine vidnux.

## Root cause

`handleAigenAsset` in `package-engine-server.js` resolved every
`/aigen-assets/<path>` request with `path.resolve(VIDNAS_AIGEN_ROOT, raw)` — the
hard-coded VIDNAS constant — while every other aigen/package route resolves
through `aigenPaths(options)` (env-overridable via `AIGEN_VIDNAS_ROOT` /
`AIGEN_SCRIPT_PACKAGES`, the pattern the repo's own tests use). With fixture
env overrides set, API routes read the fixture root but asset requests still
hit VIDNAS → 404 → broken-image placeholders in 4 cockpit panels
(image selector, I2V workspace, video review, resolve handoff) and no way for
any test to ever cover asset serving.

A second wrinkle surfaced during verification: asset URLs come in two shapes —
`script-packages/<pkg>/<rel>` (project pages) and bare `<asset-folder>/<rel>`
(package-run media). Since `AIGEN_SCRIPT_PACKAGES` can legitimately point
outside the aigen root, the two shapes need separate roots.

## Fix (narrow)

`handleAigenAsset(req, res, url, options)` now:

- resolves via `aigenPaths(options)` (call-time env reads, same as all routes);
- maps `script-packages/<pkg>/…` onto `paths.scriptPackages` (so a detached
  `AIGEN_SCRIPT_PACKAGES` override works) and all other paths onto
  `paths.aigenRoot`;
- keeps both traversal guards (segment filter for `..`/dot-prefixed parts, plus
  resolved-path containment against the chosen root);
- keeps clean 404 for missing files, same URL shapes, same MIME/caching.

Call site passes `serverOptions` (also enables programmatic
`createServer({aigenRoot})`). **With no env overrides both branches resolve to
exactly the previous VIDNAS paths — production behavior is unchanged.** No
schema, UI, or layout changes.
