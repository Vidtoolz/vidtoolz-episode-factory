## 4. Pipeline contract (verified in code at f8b9c60)

```
shot-plan.json ──(already generated, v0.2)──▶ earth-studio.esp   [UNTOUCHED]
     │                                              │
     │                              manual Earth Studio import + frame export
     ▼                                              ▼
earth-studio/frames/*.png|jpg|jpeg  ◀── operator copies export here
     │
     ▼  POST /api/earth-studio/render     (earth-studio-lane.js startRender)
ffmpeg -framerate <job.frame_rate> -pattern_type glob -i frames/*.<ext>
       -c:v libx264 -pix_fmt yuv420p -movflags +faststart renders/<slug>.mp4
     │
     ▼  POST /api/earth-studio/stage      (earth-studio-lane.js stageToVidnas)
/mnt/vidnas_public/VIDTOOLZ/99_SANDBOX/earth-studio-pilot/<package-id>-<slug>.mp4
```

Guards and refusal conditions (all in `earth-studio-lane.js` +
`package-engine-server.js`):
- Every POST requires the local write nonce (`validateLocalWriteRequest`:
  local Host + local Origin + nonce from `GET /api/aigen/status` →
  `localWriteNonce`, header `x-local-write-nonce` or payload field).
- Project id resolution (`resolveAigenPackageDir`): id must match
  `^[A-Za-z0-9._-]+$`, no traversal, must resolve INSIDE the aigen
  script-packages root, must exist. 404/400 otherwise.
- `startRender` refuses: an already-active render (409, single-slot job),
  a missing job.json (400), an empty frames dir (400). Completed job state
  is held 60 min for audit, then cleared. Output overwrites a previous
  render of the same slug (`ffmpeg -y`) — re-render is safe but replaces.
- `stageToVidnas` refuses a missing MP4 (400) and any stage dir matching the
  approved-media pattern (`/v\d+-approved|v1-approved|03_SHARED_MEDIA_LIBRARY\/.*approved/i`).
  It copies with `copyFileSync` — an existing staged file of the same name is
  silently REPLACED (audit note B4); hash both sides (§6) to prove identity.
- Failure behavior: ffmpeg non-zero exit → job `exit_state: "failed"` with
  stderr tail in `GET /api/earth-studio/job-status`; a partial MP4 may remain
  on disk — re-running the render overwrites it cleanly.

The checker is read-only (repo script-safety header, guard-tested), refuses
package dirs outside `package-runs/` and the aigen script-packages root,
refuses symlinked frames, treats "not run yet" as PENDING (exit 0) and
contradictions as FAIL (exit 1): frame-count mismatch, sequence gaps,
duplicate numbers, mixed extensions (the lane would silently ignore some),
non-uniform dimensions, MP4 fps/duration/frame-count drift (ffprobe),
staged-copy hash mismatch, staging containment breach, approved-media path.

Frames → MP4 via API (Option 1), exactly:

```bash

  -H 'Content-Type: application/json' -H "x-local-write-nonce: $NONCE" \
  -d '{"id":"b2-earth-studio-proof-20260721"}'
curl -s http://127.0.0.1:8010/api/earth-studio/job-status   # poll until exit_state completed
```

## 9. Failure-classification branches (documented, NOT implemented)

- **A — import clean, motion correct:** proceed (export → MP4 → stage →
  evidence). No `.esp` changes are justified.
- **B — Earth Studio rejects the file:** only then export the SMALLEST real
  Earth Studio-saved project from the operator's account and classify:
  container/syntax vs schema version vs missing required fields vs value
  types vs keyframe structure vs project metadata. No guessing without the
  real reference file.
- **C — imports, geography drifts:** evidence first; then inspect coordinate
  order, units, position-vs-target, altitude reference, interpolation
  semantics, projection, Earth Studio normalization. Assume no single cause.
- **D — geography right, motion wrong:** keyframe times, timeline units,
  heading/tilt/roll/FOV, easing, interpolation, target semantics.
- **E — `.esp` import cannot be made reliable:** document Point-to-Point
  Quick Start (Earth Studio's own geocoding — type "London", never blind
  numeric entry) as the supported manual seam; stop claiming generated-`.esp`
  execution support unless later proven. The untouched-import evidence stays
  mandatory even if the fallback succeeds.
