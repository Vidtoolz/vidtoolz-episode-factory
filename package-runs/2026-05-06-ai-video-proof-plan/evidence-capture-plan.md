# Run 3 Evidence-Capture Plan

Status: planning only

## 1. Purpose

Define the durable evidence required before any shooting approval for Run 3.

The proof-plan video depends on showing or preserving what the AI actually produced. A pasted transcript is useful planning evidence, but it is insufficient alone for production proof.

## 2. Evidence Boundary

- Current durable evidence: pasted ChatGPT transcript only.
- Missing durable evidence: screenshot or screen-recording evidence.
- Required before shooting approval: durable screenshot or screen-recording evidence that preserves or shows the actual AI-produced output relevant to the proof-plan video.
- The captured evidence must clearly identify whether the output came from ChatGPT, another AI tool, or a prepared/stress-test example.
- ChatGPT did not generate `10 AI Tools Every Creator Should Try in 2026`.
- `10 AI Tools Every Creator Should Try in 2026` must be treated only as prepared/stress-test material unless future durable evidence proves otherwise.

## 3. What Must Be Captured

Capture enough visual evidence to support the claims the video intends to make about the proof-plan workflow.

Minimum target content:

- The AI tool or prepared/stress-test context being reviewed.
- The actual generated or reviewed output visible on screen.
- Enough surrounding context to make clear what produced the output.
- A visible timestamp or file metadata trail where practical.
- A clear distinction between:
  - ChatGPT-produced output.
  - Output from any other AI tool.
  - Prepared/stress-test examples created for evaluation.

## 4. Minimum Acceptable Evidence

At least one of the following is required before any shooting approval:

- One durable screenshot showing the relevant AI-produced output and enough context to identify the source tool.
- One durable screen-recording showing the relevant AI-produced output and enough interaction/context to identify the source tool.
- Multiple screenshots if one image cannot show both the output and the source context clearly.

The minimum evidence must be stored in the Run 3 folder and referenced by an evidence index before it can be used for a shooting approval decision.

## 5. Optional Stronger Evidence

Stronger evidence may include:

- A short screen recording from prompt/context through visible output.
- A screenshot set showing prompt/context, generated output, and tool identity.
- A local evidence index describing what each capture proves and what it does not prove.
- Redacted duplicate captures if privacy-sensitive material appeared in the original UI.

## 6. What Must Not Be Claimed

- Do not claim a pasted transcript is sufficient production proof.
- Do not claim a screenshot exists until it is captured and stored.
- Do not claim a screen recording exists until it is captured and stored.
- Do not imply ChatGPT generated `10 AI Tools Every Creator Should Try in 2026`.
- Do not claim prepared/stress-test material is captured AI output.
- Do not claim shooting, editing, publishing, upload prep, final title/thumbnail, project-state promotion, commit, push, or Hermes brain write is approved.

## 7. Privacy/Safety Checks Before Capture

Before any screenshot or screen-recording capture:

- Close unrelated tabs and apps.
- Hide or remove private data.
- Do not capture emails, account details, tokens, billing pages, private chats, unrelated browser clutter, personal information, or client/project material outside Run 3.
- Use a clean browser window or isolated workspace where practical.
- Redact or retake the capture if private or unrelated information appears.

## 8. File Naming Convention

Use descriptive local filenames:

- `evidence/screenshot-YYYY-MM-DD-short-description.png`
- `evidence/screen-recording-YYYY-MM-DD-short-description.mp4`
- `evidence/evidence-index.md`

Examples:

- `evidence/screenshot-2026-05-07-chatgpt-proof-plan-output.png`
- `evidence/screen-recording-2026-05-07-proof-plan-output-context.mp4`

Do not create the `evidence/` folder or evidence files until capture is explicitly approved later.

## 9. Storage Location Inside Run 3 Folder

Suggested storage location:

- `package-runs/2026-05-06-ai-video-proof-plan/evidence/`

Suggested index:

- `package-runs/2026-05-06-ai-video-proof-plan/evidence/evidence-index.md`

The evidence index should list each screenshot or screen-recording file, source tool, captured output, date captured, privacy check result, and what the evidence proves.

## 10. Verification Checklist After Capture

After evidence is captured, verify:

- The screenshot or screen-recording file exists in the Run 3 `evidence/` folder.
- The evidence index exists.
- The evidence index identifies the source as ChatGPT, another AI tool, or prepared/stress-test material.
- The capture shows or preserves the actual AI-produced output relevant to the proof-plan video.
- The capture does not expose private data, unrelated browser clutter, account details, emails, tokens, or personal information.
- The capture does not imply ChatGPT generated `10 AI Tools Every Creator Should Try in 2026`.
- The capture does not approve shooting, editing, publishing, upload prep, final title/thumbnail, project-state promotion, commit, push, or Hermes brain write.
