## 2. Exact Prompt

```text
Give me 10 YouTube video ideas for solo creators using AI tools in their video workflow. Make the ideas practical and current.
```

## 3. Manual Capture Instructions

1. Open the ChatGPT web UI in a normal logged-in browser session.
2. Start a new chat or clearly separate this test from unrelated work.
3. Paste the exact prompt from this pack.
4. Submit the prompt without adding extra instructions.
5. Wait until ChatGPT finishes responding.
6. Copy the full visible response text.
7. Paste the copied response into `capture-transcript.md` under a clearly dated capture section.
8. Record whether any screenshot or screen recording was saved as durable evidence.
9. If no durable screenshot or recording was saved, say that plainly.
10. Do not summarize from memory. Paste the actual output first, then write the review notes.

## 5. How To Identify The Closest Weak Idea

After pasting the actual output, inspect each idea and ask:

- Is it a broad listicle or generic AI-tools roundup?
- Does it lack a clear proof/demo moment?
- Does it lack a practical production path?
- Does it rely on novelty rather than viewer payoff?
- Is it close to the prepared weak example `10 AI Tools Every Creator Should Try in 2026`?

Record the closest weak idea exactly as ChatGPT wrote it. If no weak idea is close enough, say so. Do not force a match.

## 6. How To Decide Whether `final-script.md` Needs Revision

Review `final-script.md` after the transcript is pasted.

The script likely needs revision if:

- it claims ChatGPT generated an example that the transcript does not contain
- it treats a prepared weak example as captured output
- it depends on a screenshot or recording that was not saved
- it repeats ChatGPT tool-capability claims as facts without separate verification
- the best captured idea changes the proof-plan framing

The script may remain valid if:

- it clearly labels prepared examples as prepared
- it uses the captured output only for claims actually supported by the transcript
- it keeps the packaging candidate marked as candidate, not final title
- it keeps title/thumbnail approval unresolved
