Step 1
Brand direction: "Show craft, don't explain frameworks." Pick a topic where you can demonstrate something concrete on screen — a real workflow, a real tool result, a real editing technique. Not abstract theory.
Key question before scripting: "What will this video prove on screen?" If the answer is vague, fix it before writing the script. The May 6 video failed because this was asked too late.
1 Pick a Topic Hermes assists

Browse system-generated candidates or submit your own topic idea. The system reviews it against 7 brand criteria (AI relevance, authority fit, visual potential, specificity, one-claim pattern, trust risk, abandoned-topic echo). This is an assessment, not an approval.

  • Open Topic Scout — browse 25 candidates (left column) or type your own idea (right column)
  • Your submitted topic auto-appears in the Package Engine as a candidate card
  • Open Package Engine, review all candidates, click "Select winner" on your chosen one
  • Click "Confirm and Save" in the confirmation panel — this writes selected-package.json and selected-package.md to the run folder
Do not write any script before topic selection is done. The Topic Scout exists to enforce this. No jumping ahead.
Open Topic Scout Open Package Engine
2 Research and Outline Hermes assists

Find 2-3 credible sources that support your claim. Use ChatGPT or Hermes web_search. Do not invent sources.

  • Save sources and key quotes in research-pack.md in the run folder
  • Write research-sufficiency-review.md: is there enough proof to proceed? If thin, stop and pick a different topic
  • Use the Package Engine outline prompt handoff. It shows the prompt, copies it, and opens ChatGPT or Claude.
  • Save output as outlines.md. Choose the best outline. Save as final-outline.md
Note: There is no cockpit page for the research stage. You create research-pack.md and research-sufficiency-review.md manually in the run folder. For the outline prompt, use Package Engine's "Open ChatGPT" or "Open Claude" buttons.
Open Package Engine Handoff Package Engine copies the outline prompt, then opens ChatGPT/Claude for you.
3 Write the Script Hermes assists

Draft the script using the proven pattern:

  • ONE claim + ONE concrete real-life example + ONE clear point
  • Conversational "explain to a friend" tone with humor
  • Under 250 words (target 1-3 minutes spoken, ~390-450 words at 130-150 wpm max)
  • Mark presenter segments with [A-ROLL] and generated visuals with [B-ROLL]

Save as script-draft.md. Get a second opinion from ChatGPT or Hermes. Revise and save as final-script.md. Read aloud and time it — if over 3 minutes, cut.

Open ChatGPT Open Claude No prompt is auto-copied here. Paste your script-draft.md manually for review.
4 Claims Check Hermes assists
  • List every factual assertion in final-script.md
  • Cross-check each against research-pack.md sources
  • Flag or remove unsupported claims
  • Record the check result in STATUS.md in the run folder
Do not proceed if any claim is unsupported. Either find a source, reword as opinion, or cut the claim.
5 Packaging (Draft) Hermes assists
  • Write 3-5 title candidates — curiosity-driven, under 60 characters
  • Describe thumbnail concept in thumbnail-mockup-notes.md
  • Set target duration, aspect ratio (9:16), platform in package metadata
  • Save in selected-package.md and youtube-package.json
Do NOT finalize title/thumbnail yet. That happens at the Publish Gate (Step 12). This is draft only.
6 Image Prompts Hermes assists

Each [B-ROLL] tag in the script needs a visual. Write FLUX prompts for each beat.

  • Open the Image Prompts Editor
  • Each prompt: subject, style, mood, composition for 9:16 vertical
  • Photorealistic only. NO text in images. NO script narration in prompts (causes rendered text).
  • Add anti-text prefix at the start of each prompt
  • Save image-prompts.json in the run folder
Open Image Prompts Editor
7 Image Generation Hermes assists

Two paths — pick one:

  • vidnux local FLUX (faster, ~48s/image): Launch ComfyUI on vidnux. Run generate-missing-prompts.py. Crashes after ~33-50 gens (auto-restarts).
  • PRESTO ComfyUI (http://192.168.50.187:8188): Use FLUX workflow there. Same prompts, same output resolution (1080x1920).

Inspect each image: matches script beat? Is it 9:16? Quality acceptable? If wrong, adjust prompt and regenerate.

9 Video Generation Hermes assists

Two paths — pick one:

  • PRESTO Wan2.2 I2V (default, ~12 min/clip): Queue selected images through ComfyUI on PRESTO. Settings: 81 frames, ~2.7 seconds. Do NOT use 149 frames (OOM timeout).
  • Kling web UI (manual alternative): Use the SAME images and SAME prompts as Wan2.2. Generates 8-second clips. Good for shots needing longer motion.
CRITICAL: Rename ALL generated media to ASCII-safe hyphenated names before Resolve import. Examples: kling-setB2-03.mp4, wan-scene01-81f.mp4. No Chinese characters, spaces, or parentheses. Kling outputs 1080x1916 (4px short of 1920) — needs 4px crop/pad in Resolve.

Move staged MP4 files to VIDNAS: /mnt/vidnas_public/VIDTOOLZ/aigen/image-to-video/production/wan22-81f/

10 A-Roll Recording Hermes assists

Record yourself on camera with green screen. Vertical 9:16 framing.

  • Set up OBS on vidnux: green screen, camera, microphone
  • Open final-script.md. Identify all [A-ROLL] segments
  • Record presenter segments — one at a time, take by take
  • Save recordings to VIDNAS: /mnt/vidnas_public/VIDTOOLZ/camera_originals/
  • Tell Hermes which take to move into the active run subfolder (be explicit)
  • Hermes runs ffprobe inspection — review resolution, frame rate, duration, codec
  • Approve the take or re-record. Do not keep bad takes.

If the browser protocol button does not launch OBS, open OBS manually from the desktop or run obs & in a local shell.

11 Assembly Edit in Resolve Mikko's domain

On PRESTO or vidnux, open DaVinci Resolve.

  • Create a new project. Import A-roll from VIDNAS (do not copy locally — use VIDNAS as media source)
  • Import B-roll from VIDNAS (staged MP4 clips from the aigen lane)
  • Cut A-roll to script timing
  • Place B-roll over marked [B-ROLL] sections
  • Fix Kling 1916px height: scale or crop to 1920
  • Add green screen keying — remove green background from A-roll
  • Place generated backgrounds behind presenter where needed
  • Color match A-roll and B-roll
  • Set audio levels: voice peaking around -6dB, music/ambience under -20dB
  • Export as H.264 1080x1920 30fps. Save to VIDNAS, not local drive.
12 Publish Gate Gate — must pass

Watch the exported video end-to-end with no interruptions.

  • Fill rough-cut-watch-notes.md with timestamps and observations
  • Classify gate status: Ready, Needs Pickups, Needs Re-edit, or Rejected
  • If needs pickups: create pickup-list.md, go back to Step 10
  • Run title and thumbnail through packaging gate
  • Confirm all claims are still supported by research sources
Trust checklist (non-negotiable, before publishing):
  • All factual claims verifiable or marked as opinion
  • Every AI visual classified (Evidence, Illustration, Decoration, Metaphor, Simulation, Fiction)
  • Non-evidence AI visuals labeled on-screen
  • Current-event sources cited in description (date + source)
  • Any visual that could mislead → cut or relabeled
  • Thumbnail reviewed for misleading claims
Only proceed to publishing if gate is "Ready." Do not bypass the gate.
