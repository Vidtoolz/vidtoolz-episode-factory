# VIDTOOLZ Episode Factory — Comprehensive System Test & Capability Audit

You are a senior production systems auditor evaluating the VIDTOOLZ Episode Factory video production cockpit. Your goal is to stress-test every user-facing workflow, backend API, UI surface, and state management layer to identify friction, gaps, and opportunities that will empower the operator (Mikko) to use this system to its full potential.

## Context & Architecture

This is a **cockpit-first video production system** for YouTube Shorts (1–3 min vertical 9:16). The operator uses:
- **VIDNAS** (192.168.61.186, mounted at `/mnt/vidnas_public`) — shared media storage
- **PRESTO** (192.168.50.187, RTX 4090) — ComfyUI with Wan 2.2 for AI video generation (81f/2.7s proven workflow)
- **vidnux** — Linux workstation with Package Engine server on port 8010
- **DaVinci Resolve** — final editing (out of automation scope)
- **Kling** — occasional manual alternative using same FLUX images + prompts

The system manages **package runs** (video projects) through stages: Research → Outline → Script → Production Prep → Asset Generation → Ready to Shoot → (manual) Edit → Publish.

## Critical Files & Endpoints

- **Package Engine server**: `package-engine-server.js` running on `http://localhost:8010`
- **Resume page**: `resume.html` (web-based project selector with cards)
- **Dashboard**: `package-runs-dashboard.html` + `package-runs-dashboard.js`
- **API endpoint**: `GET /api/package-runs/list` (returns index JSON)
- **Run index**: `package-runs-index.json` (generated by `scripts/package-runs-index.js`)
- **Package run directory**: `package-runs/{runId}/` containing `STATUS.md`, artifacts, etc.

### 4. Video Room — Focused Workspace & Actionability

The Video Room must be **genuinely actionable**, not a passive read-only view. Evaluate:

**4.1 Prominent Next Action Card**
- Large green block with:
  - Human-readable title (e.g., "Create Outline", "Creator QA Required", "Production Check")
  - Description of *what to do* and *why*
  - Copy button for the command
  - Missing artifacts section (prerequisites to complete first)
- Only shows manual review note (yellow box) when no action is determinable

**4.2 Action Type Detection Logic**
```javascript
// Pseudocode for action detection
if (creatorQaStatus !== "PASS") → "Creator QA Required"
else if (status.includes("Research")) → "Research Phase"
else if (status.includes("Outline")) → "Create Outline"
else if (status.includes("Script")) → "Write Script"
else if (status === "Ready to shoot") → "Production Check"
else if (nextCmd exists) → "Next Action"
```

**4.3 Command Construction**
- Must use correct `runId` from URL param
- Example: `node scripts/package-run-creator-qa.js package-runs/2026-06-24-ideation`
- Copy button copies to clipboard with visual feedback ("Copied!")

**4.4 Hidden Panels (CSS)**
When `data-focused-run` is set:
- `.productions-overview-panel` — hidden
- `.system-availability-panel` — hidden
- `.capability-inventory-panel` — hidden
- `.workflow-wizard-panel` — hidden
- `.media-gallery-panel` — hidden
- `.friction-log-panel` — hidden
- `.visual-beat-map-panel` — hidden
- `.beginning-triage-panel` — hidden
- `.current-focus-panel` — hidden

**Expected:** Only Video Room panel visible, max-width 1200px, centered, dominating viewport.

**Failure modes:**
- Noise panels still visible → CSS selectors incomplete or specificity issue
- Next action card shows raw command without context → action type detection broken
- Missing artifacts section empty → `run.missingExpectedArtifacts` not populated
- Copy button doesn't work → JS event handler missing or clipboard API blocked

**8.4 Invalid `?run=` Parameter**
